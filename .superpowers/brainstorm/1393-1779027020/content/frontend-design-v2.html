Key behaviours

  • Group chats section appears first (department, mission, custom), sorted by most recent activity
  • DM section appears below with its own "+" button
  • Tapping an existing DM → opens it directly (same ChatDetailScreen)
  • Tapping "+" → opens DM picker (users grouped by department)
  • Tapping a person in the picker → POST /api/chats/direct → navigate to chat
  • If DM already exists, the same chat is returned — no duplicates
  • AppBar shows the other person's name + role + department
  • Existing ChatDetailScreen reused — settings/members button hidden for DMs
